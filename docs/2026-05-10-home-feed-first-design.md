# 首页 Feed-first 调整设计

## 1. 背景

首页在移除大头部后，仍保留了一块 `arcade-bar`，包含“作品广场 / 我的作品”、作品指标和大号“创建”按钮。它继续占用首屏空间，并且与左侧导航、移动端底部导航、Feed 标题和“新建作品”入口重复。

## 2. 决策

采用 Feed-first 方案：删除整块 `arcade-bar`。

- 首页顶部直接从 Feed 标题开始。
- “全部作品 / 我的作品”的上下文由 Feed 标题表达。
- “我的作品”入口继续由左侧“工作室”和移动端底部“工作室”负责。
- 创建入口保留 Feed 标题右侧的轻量“新建作品”链接；空状态仍保留主按钮。
- 首页不再展示作品数、试玩数等平台指标。

## 3. 用户影响

- 进入首页后第一眼看到内容，而不是控制台式指标。
- 页面减少重复导航，用户路径更直接。
- 公开 Feed 和个人工作室的边界仍保留，但放回主导航体系，而不是在内容区重复一次。

## 4. 不做

- 不新增搜索、排序、标签筛选。
- 不改 `/?tab=mine` 的兼容路径；工作室导航仍可进入“我的作品”视图。
- 不改列表数据源、卡片、详情页和生成链路。

## 5. 验证

- 首页不再出现 `arcade-bar` 控制区。
- 桌面端首屏直接看到 Feed 标题和卡片。
- 移动端无顶部重复创建按钮，底部导航不遮挡 Feed 的主要操作。
- 运行 `npx prisma generate`、`npm run lint`、`npm run build`，再启动本地服务检查页面。
