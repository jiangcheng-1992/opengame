# 一创 AI 美术增强设计

## 背景

当前生成稳定性已经优先解决，但一创产物经常停留在“可玩原型”：背景空、角色和障碍物像占位图形、UI 缺少游戏感。封面图已经在发布阶段生成，能改善 Gallery，但不能改善 iframe 里的实际试玩画面。

本设计只增强“一创”入口，不改变继续修改链路，不影响默认生成路径。

## 目标

- 在一创确认 Brief 时提供“AI 美术增强”开关，默认关闭。
- 开启后额外生成两张游戏内资产：背景图和核心图集。
- 封面图沿用现有发布阶段生成能力，不新增一次封面生图调用。
- OpenGame 必须把游戏内资产用于试玩画面；资产失败时降级为同风格的 Canvas/CSS 程序化美术，不让可玩生成失败。

## 非目标

- 不做多轮选图、局部重生或人工挑图。
- 不做完整动画帧、UI 皮肤包或素材商城。
- 不改内置精选游戏。
- 不改变 READY 的可玩性标准；视觉质量验收另起后续版本。
- 不新增登录、付费额度或用户资产管理页面。

## 用户体验

确认 Brief 卡片中增加一个独立开关：

> AI 美术增强  
> 额外生成背景和角色图集，让游戏更像完成作品。会增加生成时间。

默认关闭。用户开启后再点击“生成可玩版本”，本次一创任务会先生成美术资产，再启动 OpenGame。

任务侧栏展示当前是否开启增强，避免用户不知道为什么等待变长。

## 生图策略

开启增强后最多新增 2 次生图调用：

| 资产 | 用途 | 规格 | 要求 |
|---|---|---|---|
| 背景图 | 游戏内全屏/主舞台背景 | 16:9 | 无文字、无 UI、留出玩法空间 |
| 核心图集 | 主角、敌人/障碍、道具、目标、特效参考 | 1:1 | 6-8 个元素、统一风格、网格排列、不要文字 |

封面图继续由现有 `generateCoverImage` 在发布阶段生成。增强开启时，只强化 `coverPrompt`，让封面和游戏内资产共享同一套 Art Direction。

## 视觉模板

第一版内置 4 套模板，用关键词从 Brief 自动选择，不让用户额外决策：

| 模板 | 适用 | 风格 |
|---|---|---|
| Neon Arcade | 太空、射击、躲避、反应 | 深色背景、霓虹描边、粒子、发光 HUD |
| Pixel Quest | 平台、冒险、收集、地牢 | 像素风、瓦片感、复古 HUD |
| Cozy Handdrawn | 经营、休闲、解谜、养成 | 手绘感、暖色、圆润角色、柔和 UI |
| Toy Boardgame | 棋盘、策略、塔防、卡牌 | 桌游质感、清晰格子、棋子和卡片 |

## 数据流

1. 前端发送 `brief`、`visibility`、`artEnhancementEnabled`。
2. 后端校验参数。
3. 后端生成普通展示元数据。
4. 若开启增强：
   - 根据 Brief 选择视觉模板。
   - 生成背景图并上传到 Blob。
   - 生成核心图集并上传到 Blob。
   - 拼装增强版 OpenGame prompt，包含资产 URL 和使用要求。
   - 强化 `coverPrompt`，但不立即生成封面。
5. 创建 `Job`，`Job.prompt` 保存实际交给 OpenGame 的 prompt。
6. 启动现有 GitHub/local worker 或 Sandbox 生成链路。
7. 发布阶段继续生成封面图并写入 `Game.coverUrl`。

## 失败策略

- 背景图失败：继续生成图集；prompt 要求用程序化背景兜底。
- 图集失败：继续使用背景；prompt 要求用 Canvas/CSS 绘制角色、敌人和道具。
- 两张都失败：仍继续生成可玩游戏，但 prompt 明确要求按模板做程序化美术。
- 封面失败：沿用现有行为，不阻塞 READY。

增强失败不能用假资产冒充成功；系统消息记录本次是否开启增强和实际可用资产。

## OpenGame Prompt 要求

增强版 prompt 必须包含：

- 原始游戏 Brief。
- 被选中的视觉模板。
- 可用资产 URL。
- 明确要求使用背景图作为游戏内背景。
- 明确要求使用图集作为角色/敌人/道具视觉来源或参考。
- 禁止空白背景、纯色方块、默认浏览器按钮和无反馈碰撞。
- 如果资产加载失败，必须以同一风格程序化绘制替代。

## 验证

基础验证：

- `npm run lint`
- `npm run build`
- `npx prisma generate`

UI 验证：

- 打开 `/create`，确认开关默认关闭。
- Brief 收束后切换开关，确认请求体带上 `artEnhancementEnabled`。
- 桌面和移动布局不重叠。

真实冒烟：

- 需要生产 MiniMax/Vercel Blob 凭据。
- 开启增强生成 2-3 个游戏，确认 Blob 中有 `art/background.png` 和 `art/spritesheet.png`。
- 进入详情页试玩，确认画面实际引用或融合了资产。
