# 游戏公开/私密切换设计

## 背景

作品在创建时已经支持 `PUBLIC` / `PRIVATE` 可见性，但生成完成后作者不能调整。用户需要在试玩和继续修改阶段，把公开作品改为私密，或把私密作品重新公开。

本次只实现作品可见性切换，不调整广场 feed 排序规则。

## 产品规则

- 入口放在 `/games/:id/edit` 顶部状态区，和“当前版本状态”同层展示。
- 只有作品作者可以切换公开/私密。
- `READY`、`FAILED`、以及已有可玩版本的 `GENERATING` 作品允许切换。
- 切换为私密后，作品从公开广场消失；非作者访问详情或文件代理继续返回不可见。
- 切换为公开后，只有 `READY` 作品会进入公开广场；非 READY 作品仍受现有广场条件限制。
- 切换可见性不触发重新生成，不影响旧版本试玩，也不影响正在进行的新版本任务。

## 交互设计

- 顶部右侧新增“公开 / 私密”双态控件。
- 控件展示当前影响：
  - 公开：会出现在广场
  - 私密：仅自己可见
- 保存中禁用按钮并显示“保存中”。
- 保存失败在控件附近展示错误，不复用生成任务错误，避免让用户误以为生成失败。

## 技术设计

- 新增 `updateVisibilitySchema`，只接受 `PUBLIC` / `PRIVATE`。
- 在 `PATCH /api/games/:id` 更新 `Game.visibility`。
- 接口先校验匿名作者身份，再校验作品是否处于编辑工作台允许的状态。
- 编辑页服务端把 `visibility` 传入 `EditGameWorkbench`。
- 前端更新成功后刷新路由数据，保证“我的作品”和详情权限状态与服务端一致。

## 验证

- 基础验证：`npm run lint`、`npm run build`、`npx prisma generate`。
- 手动验证建议：
  - 作者在编辑页把 READY 作品从公开切为私密，广场不再展示。
  - 作者仍能从“我的”进入编辑页。
  - 再切回公开后，READY 作品重新满足广场展示条件。
