# 首页紧凑控制条调整设计

## 1. 背景

首页已经改为单一平铺 Feed 后，原来的大头部区域继续展示“可玩作品流”“首页”和一段说明文案。它没有提供筛选、发现或创作决策价值，还重复了左侧导航、作品广场 tab 和 Feed 标题，占用了首屏卡片空间。

## 2. 决策

采用方案 A：删除大头部，只保留一个紧凑控制条。

- 移除首页大标题、eyebrow 和长说明文案。
- 将“创建”按钮移动到 `arcade-bar` 内，和“作品广场 / 我的作品”、作品指标组成一个操作区域。
- Feed 区块直接接在控制条下方，让首屏更快露出游戏卡片。
- 移动端顺序为：创建按钮、tab、指标、Feed。

## 3. 用户影响

- 用户进入首页后更快看到作品内容，而不是先读一段重复说明。
- “创建”仍然是高优先级动作，但不再占用一整块 hero 区域。
- 当前 MVP 内容量较少时，页面更像可浏览的作品流，而不是空泛的平台首页。

## 4. 不做

- 不新增搜索、标签筛选或推荐算法。
- 不改“作品广场 / 我的作品”的信息边界。
- 不改卡片数据、生成链路和详情页行为。

## 5. 验证

- 首页不再出现大号“首页”标题区域。
- 桌面端控制条一行承载 tab、指标和创建按钮。
- 移动端创建按钮在控制条顶部，Feed 无横向滚动。
- 运行 `npx prisma generate`、`npm run lint`、`npm run build`，并打开本地首页检查。
