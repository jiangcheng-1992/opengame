# 首页封面优先信息流优化设计

## 1. 背景

首页作品卡片当前把标题、作者、日期、标签和 Like 拆成多行放在封面下方。信息完整，但在浏览场景里下方白色区域过高，削弱了游戏封面的吸引力，也让高分辨率屏幕下的信息流显得不够丰满。

## 2. 决策

采用封面优先的游戏货架卡片。

- 标题移动到封面底部，通过渐变遮罩保证可读性。
- 播放数继续保留在封面右下角，和标题错位展示。
- 封面下方压缩为一条短信息栏，只展示 1-2 个标签和 Like 数；没有标签时用日期兜底。
- 内置精选、草稿、生成中等状态标签继续保留在封面左上角。
- 首页公开 Feed 在大屏放宽内容容器，并使用 `auto-fill + minmax` 自动增加列数，避免超宽屏只显示少量卡片。

## 3. 用户影响

- 用户进入首页后第一眼看到的是游戏画面和标题，而不是大块元信息。
- 单张卡片高度降低，首屏能露出更多作品。
- 高分辨率屏幕下右侧信息流更饱满，减少大面积空白。
- Like、播放数、标签仍可辅助判断，但不再喧宾夺主。

## 4. 不做

- 不新增推荐算法、筛选器或排序逻辑。
- 不改作品数据结构、生成链路和详情页行为。
- 不把作者、日期等完整元信息做成首页强展示；需要完整上下文时进入详情页。

## 5. 验证

- 首页卡片标题显示在封面底部，封面下方信息栏明显变短。
- 桌面、超宽屏、平板和移动端无横向滚动，卡片文字不重叠。
- 运行 `npx prisma generate`、`npm run lint`、`npm run build`。
- 启动本地页面检查首页桌面和移动布局。
