# 创建页随机游戏灵感设计（已移除）

## 当前状态

2026-05-10 已按产品决策移除：新建作品开始时不再自动随机出题，也不再提供“换一组 AI 灵感”。创建页回到用户自由输入后再进入流式头脑风暴的主链路。

移除原因：随机出题会替用户预设创作方向，弱化“用户先表达想法，系统再澄清玩法、操作、目标和视觉风格”的核心流程。

## 背景

创建页的空状态按钮原本是固定玩法分类，例如“经典 RPG”“实时操作”“放置/自动战斗”。这不能帮助还没输入 prompt 的用户决定“做什么游戏”，反而提前把问题带到玩法参数选择上。

## 原方案（已废弃）

- 新增 `GET /api/games/brainstorm/ideas`，使用 MiniMax 文本模型实时生成 3 个随机游戏灵感。
- 每个灵感必须是可直接点击发送的完整方向，至少包含题材、核心操作或胜负目标，优先带视觉风格。
- `/create` 在没有历史对话且没有初始 prompt 时自动拉取一组灵感，并提供“换一组 AI 灵感”。
- MiniMax 未配置或输出不可解析时展示真实错误，用户仍可自由输入，不使用假 AI 结果冒充完成。
- 头脑风暴系统提示词补充约束：用户没想法时，建议按钮必须给具体游戏创意，不能只给抽象玩法标签。

## 原用户影响

用户第一次进入创建页时，可以直接从 AI 随机给出的游戏方向开始，而不是先理解玩法分类。点击任意灵感后，仍然进入既有的槽位澄清流程，保证最终 brief 包含玩法、操作、目标和视觉风格。

## 原验证

- `npx prisma generate`
- `npm run lint`
- `npm run build`
- 有 `MINIMAX_API_KEY` 时，打开 `/create` 验证初始灵感是否由接口随机生成，并可点击进入对话。
