# 沉浸式游玩页改造设计

## 1. 决策

本轮把详情页从“作品信息页里嵌入游戏”调整为“以试玩为主的播放器页面”。

- 默认体验优先试玩，游戏 iframe 是首屏主体。
- 保留全局桌面侧栏和移动底栏，不做全站外壳切换。
- 不新增全屏按钮、抽屉、快捷键或额外播放器模式。
- Like 保留在游戏下方；继续修改从公共详情页移出，改由“我的作品 / 创作台”进入 `/games/:id/edit`。

## 2. 信息层级

顶部只保留必要识别信息：

- 返回作品广场。
- 游戏标题。
- 当前状态。
- 播放和喜欢数据。

摘要、标签和内置说明放在游戏下方。生成日志、创作对话历史和作者修改入口不属于公共游玩页。

## 3. 布局

桌面端在保留全局导航的前提下，让内容区尽量给游戏画面：

- 游戏舞台独占一整行，不与操作面板并排。
- iframe 使用稳定的宽高比例和更高的可视高度，减少首屏信息噪声。
- 游戏下方先放 Like 操作面板，再放作品信息和必要辅助说明。

移动端采用单列：

- 顶部紧凑栏自动换行。
- iframe 保持 16:10 左右比例，避免横向滚动。
- 下方表单和记录自然纵向展开，底部导航不遮挡内容。

## 4. 不做

- 不改数据库结构、API、生成链路和同源播放代理。
- 不把 Blob URL 直接放回 iframe。
- 不新增假数据、评论、登录、分享或运营入口。
- 不调整首页、创建器和全局导航的产品结构。

## 5. 验证

- `npx prisma generate`
- `npm run lint`
- `npm run build`
- 本地打开 `/games/builtin-neon-bricks` 检查桌面和移动布局：
  - 首屏游戏优先。
  - 全局导航仍保留。
  - Like 操作面板在游戏下方。
  - 不展示生成日志和创作对话历史。
  - 页面无横向滚动。
