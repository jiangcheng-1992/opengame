# 轻量任务侧栏设计

## 背景
- 当前一创右侧同时承担需求进度、完整 Brief、可见性和生成进度，信息密度偏高。
- 当前二创右侧同时展示修改计划、对话、完整修改 Brief、生成按钮和进度，用户会被中间过程打断。
- 新目标是让工作台分工更清楚：左侧负责创作/试玩与确认，右侧只负责当前任务状态。

## 决策
- 一创和二创右侧都改为轻量任务侧栏，不再展示完整 Brief。
- 一创的 Brief 确认放回左侧对话流，用确认卡完成生成前确认。
- 二创未明确修改意图前，右侧只展示轻量修改输入框和少量快捷方向。
- 二创提交修改意图后，右侧切换为任务卡，展示任务信息、生成进度、结果状态。
- 一创和二创都保留运行日志查看能力，但默认折叠；点击后展开详细日志，便于定位真实生成问题。

## 信息架构

### 一创页面
- 左侧主区：
  - 流式创作对话。
  - 可点回复。
  - 当需求收束后，在对话流底部展示“确认 Brief”卡。
  - 确认卡包含四个必要信息摘要、公开/私密选择、生成可玩版本按钮。
- 右侧侧栏：
  - 任务：新游戏创作。
  - 状态：准备中 / 等待生成 / 生成中 / 已完成 / 失败。
  - 进度：确认需求、创建任务、生成游戏、自动试玩、发布结果。
  - 结果：等待生成、可试玩版本、生成失败原因。
  - 运行日志：默认折叠，生成启动后可展开查看详细日志。

### 二创页面
- 左侧主区：
  - 当前版本试玩 iframe。
  - 失败作品展示失败原因。
  - 旧版本在新版本生成期间继续可玩。
- 右侧侧栏：
  - 未提交修改意图前：轻量修改意图框，支持快捷修改方向。
  - 提交后：任务卡展示“修改当前游戏”、状态、进度、结果。
  - 运行日志默认折叠，展开后显示详细日志和错误原因。

## 交互规则
- 右侧不展示完整一创 Brief 或二创 Brief。
- 一创只有在 Brief 确认卡中点击生成，才启动真实生成链路。
- 二创输入少于 4 个字符时不允许提交，避免空泛修改。
- 生成中禁用重复提交。
- 修改生成失败时，旧版本继续可玩，右侧任务卡显示失败并允许用户调整后再试。
- 一创和二创运行日志都不是默认主内容，只有用户主动展开时显示。

## 不做
- 不新增数据库表。
- 不新增版本历史 UI。
- 不恢复 Remix / 二创入口。
- 不把运行日志默认展开，不把日志做成主要内容。
- 不改生成链路、自动试玩验证和同源 iframe 代理。

## 验证
- `npx prisma generate`
- `npm run lint`
- `npm run build`
- 本地打开 `/create` 检查：
  - Brief 确认卡在左侧对话流内。
  - 右侧只展示任务信息、进度和结果。
  - 生成启动后运行日志默认折叠，点击可展开详细日志。
- 本地打开 `/games/:id/edit` 检查：
  - 未提交修改意图前右侧是轻量输入框。
  - 提交后右侧切换为任务卡。
  - 运行日志默认折叠，点击可展开详细日志。
  - 移动端无横向滚动，输入和按钮可操作。
