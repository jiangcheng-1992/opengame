# 首页单一 Feed 调整设计

## 1. 背景

当前首页把公开作品拆成“玩家精选”“最新生成”“适合二创”三段货架。这个结构适合内容量已经足够多、且推荐信号可信的平台；但 MVP 当前更需要表达“真实生成出来的作品正在进入广场”，过早分层会让页面显得内容不足且解释成本偏高。

## 2. 决策

首页公开广场改为单一平铺 Feed。

- 保留“作品广场 / 我的作品”作为顶层导航，因为这是公共发现和个人工作区的真实边界。
- 作品广场内部不再做“精选、适合二创、最新生成”等分区。
- Feed 使用 `listGames("all")` 现有创建时间倒序，不新增推荐算法。
- 卡片保持同一尺寸，不再突出首卡，避免少量内容时显得像人工运营位。

## 3. 用户影响

- 用户打开首页后直接浏览作品，不需要理解平台定义的内容分类。
- 当前内容少时页面更诚实，新增作品会自然进入同一个流。
- 继续修改和 Like 的入口仍在详情页和卡片信息中承接，不靠首页分区强行解释。

## 4. 不做

- 不新增榜单、推荐权重、搜索、标签筛选。
- 不改数据库结构和生成链路。
- 不把草稿放进公共 Gallery；草稿只出现在“我的作品”。

## 5. 验证

- 首页公开广场只出现一个 Feed 区块。
- 页面文案不再出现“玩家精选”“适合二创”“最新生成”。
- 桌面和平板为响应式网格，移动端单列，无页面级横向滚动。
- 基础验证继续运行 `npx prisma generate`、`npm run lint`、`npm run build`。
