# 统一暖色编辑器 UI 优化设计

## 1. 背景

当前产品首页和创建器共用深色街机视觉：霓虹绿、强渐变、大字号、重阴影和大量色块。这能表达“游戏感”，但对 MVP 当前两个核心任务不够合适：

- 首页是作品 Feed，用户需要快速扫卡片、判断是否点开试玩。
- 创建器是 AI Chat 工作台，用户需要专注输入、阅读 AI 追问、确认 brief。

如果继续放大街机感，两个页面都会显得吵；如果首页和创建器分别套不同风格，又会像两个产品。

## 2. 参考判断

参考 getdesign.md 中的 Claude 风格：暖色画布、珊瑚主色、低噪声 editorial layout、克制边框和少量深色产品 chrome。它更适合 AI Chat 和创作工作台。

不采用 Linear / Cursor 作为主风格，因为它们偏深色开发工具，会延续当前“黑、亮、重”的问题。不采用 Airtable 作为主风格，因为它适合结构化数据 Feed，但对 AI Chat 的产品气质不如 Claude 贴合。

## 3. 决策

采用统一暖色编辑器风格，首页 Feed 和创建器 Chat 使用同一套视觉 token。

- 背景：统一使用暖白 / 米色画布，不再使用深色全局背景。
- 主色：统一使用珊瑚色，承担主按钮、焦点、重要状态和少量链接。
- 深色：只作为导航标识、封面遮罩、播放态和局部 chrome，不作为整页底色。
- 字号：页面标题和卡片标题整体降一档；正文保持 14-16px，提高可读性但减少压迫感。
- 字重：从大量 800/900 降到 500/600，保留必要层级。
- 卡片：统一白色或暖白表面、1px hairline、轻阴影；不再使用大面积彩色背景。
- 圆角：继续使用 8px 为主，避免过度圆润。
- 动效：只保留 hover 上浮、边框变化和按钮反馈。

## 4. 首页 Feed

- Feed 仍然是首屏主体，不恢复 hero 或大控制台。
- 卡片封面保留 16:9，承载游戏视觉；卡片 body 降低色彩占比。
- 状态、标签、数据统一使用小号 pill / metadata 样式，不再用高饱和色块抢内容。
- “新建作品”保留轻量链接，不做大型 CTA 区块。

## 5. 创建器 Chat

- Chat shell 和 brief panel 使用同一套暖白卡片系统。
- 用户消息与 AI 消息只用轻微底色和边框区分，避免一边深色一边浅色导致割裂。
- 输入框使用编辑器式低噪声 composer，发送按钮使用珊瑚主色。
- 右侧生成确认面板不再像仪表盘，用轻量卡片、列表和分段控制表达进度。

## 6. 不做

- 不改数据结构、生成链路、接口和页面路由。
- 不新增搜索、筛选、运营位或登录能力。
- 不引入外部字体依赖，继续使用系统字体与 Inter fallback。
- 不添加 mock 内容。

## 7. 验证

- 首页和创建器在同一背景、主色、边框、卡片规则下呈现。
- 首页 Feed 首屏仍直接露出卡片。
- 创建器输入、建议、可见性切换、生成按钮仍可操作。
- 运行 `npx prisma generate`、`npm run lint`、`npm run build`。
- 启动本地服务，打开首页和 `/create` 做视觉检查。
