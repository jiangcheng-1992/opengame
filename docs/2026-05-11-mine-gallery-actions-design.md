# 我的作品入口与分类设计

## 背景
- 侧边栏里的“创建”入口和首页/我的页里的创建入口重复，容易让用户不知道该从哪里开始。
- “工作室”命名偏重生产工具感，当前 MVP 对非技术用户更适合用“我的”承载个人作品。
- “我的作品”目前按时间平铺，草稿、生成中、完成、失败混在一起，处理优先级不够清楚。

## 设计
- 侧边栏与移动底部导航只保留“首页”“我的”，删除“创建”菜单项；创建动作通过页面主按钮进入。
- “工作室”统一改为“我的”，相关返回入口同步改文案，降低命名分裂。
- 首页作品流右侧的“新建作品”从文字链接升级为主按钮，继续跳转 `/create`。
- “我的”页顶部新增紧凑行动 Banner：
  - 左侧显示“我的作品”、一句状态说明和强化主按钮“新建作品”。
  - 右侧使用一张生成插画作为头图，主题为轻量游戏创作台，不做大面积营销海报。
  - Banner 高度控制在首屏可见作品列表的范围内。
- “我的”页 Banner 下方增加状态分类 Tab，顺序为“全部 / 创作中 / 已完成 / 待修复”。

## 分类规则
| Tab | 状态 |
| --- | --- |
| 全部 | 全部我的作品 |
| 创作中 | `DRAFT`、无可玩版本的 `GENERATING` |
| 已完成 | `READY`，包括正在生成新版本但旧版仍可玩的作品 |
| 待修复 | `FAILED` |

## 路由与数据
- 继续复用 `/?tab=mine` 作为“我的”页入口。
- 状态分类使用 URL 参数保留状态，例如 `/?tab=mine&status=active`，便于刷新、返回和分享。
- 公共首页不展示状态分类，仍只展示内置精选和公开 `READY` 作品。
- 已完成作品继续修改时，主状态保持 `READY`；卡片根据最新 Job 展示进度提示，但点击仍进入修改工作台。

## 验证
- 跑 `npm run lint`、`npm run build`、`npx prisma generate`。
- 涉及 UI 后启动本地服务并打开首页与“我的”页，确认导航、按钮、分类和头图渲染正常。
