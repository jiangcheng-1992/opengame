# 首页精选头部与顶部导航设计

## 1. 背景

首页的目标是让用户快速感知“这里有很多可玩的游戏”。单纯网格流虽然密度高，但缺少首屏焦点；左侧导航也占用横向空间，让封面卡片在宽屏下不够舒展。

## 2. 决策

- 全站主导航从左侧栏改为顶部横栏：左侧是 OpenGame 标识，中间是“广场 / 我的”，右侧只保留“创建游戏”主按钮。
- 首页公开广场顶部新增精选区域：左侧 1 个大游戏，右侧 4 个小游戏，全部来自当前真实作品列表。
- 精选区下方继续使用封面优先的信息流卡片；精选区已经展示过的前 5 个作品不在首屏下方重复出现。
- “我的作品”继续保留作品状态筛选与创作入口，不改变草稿、生成中、失败作品的流转规则。

## 3. 用户影响

- 首屏不再只是平均分配的卡片，而是先给用户一个明确的“今天先玩这个”的视觉焦点。
- 顶部导航释放横向空间，封面可以更大，游戏表达更完整。
- 创建入口保持在右上角，符合常见产品操作习惯，也避免在首页内容区重复抢注意力。

## 4. 不做

- 不新增推荐算法、排序规则、账号入口或用户头像。
- 不改变游戏数据结构、生成链路、Like 和 playCount 行为。
- 不让内置精选冒充真实生成作品，仍保留“内置精选”标识。

## 5. 验证

- 桌面端首屏出现“左大右四”的精选游戏区，下方继续是封面优先卡片流。
- 移动端精选区改为纵向结构，不出现横向滚动或文字遮挡。
- 顶部导航在广场、我的、详情/编辑路径下 active 状态正确。
- 运行 `npx prisma generate`、`npm run lint`、`npm run build`，并启动本地页面检查桌面/移动布局。
