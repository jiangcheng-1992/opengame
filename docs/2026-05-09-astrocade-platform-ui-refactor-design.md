# Astrocade 平台化 UI 重构设计

> 2026-05-10 更新：二创 / Remix 已从当前 MVP 范围移除，详见 `docs/2026-05-10-remove-remix-scope-design.md`。本文中 Remix Ready 等内容货架只作为早期探索记录，不再作为实现依据。

## 1. 决策

本轮选择方案 A：深色 Astrocade Feed。目标不是给现有页面换色，而是把第一屏从“AI 生成器工作台”改成“可试玩的游戏内容平台”。

用户先看到可以玩的作品，再进入创建和 Remix。这样更符合 OpenGame × Astrocade MVP 的本质：真实生成链路可跑以后，作品需要被发现、试玩、继续修改。

## 2. 信息结构

桌面端采用左侧常驻导航：

- 主品牌：OpenGame Arcade。
- 主按钮：Play，回到 Gallery。
- 导航：Home、Create、Studio。
- Studio 对应匿名身份下的 My Arcade，不引入登录 Profile。

首页采用横向内容货架：

- Players' Choice：按播放、喜欢、Remix 的综合信号排序。
- Trending：按创建时间展示新作品和生成状态。
- Remix Ready：只展示 `READY` 且可继续改造的作品。

移动端不使用左侧导航，保留底部 3 项导航：Home、Create、Studio，避免占用窄屏内容空间。

## 3. 视觉系统

- 深色背景承载游戏封面，降低页面 UI 对作品内容的干扰。
- 卡片封面优先，播放量和状态作为浮层，标题、作者归属和时间放在卡片下方。
- 主色使用洋红到紫色渐变，作为 Play / Create 的行动信号。
- 辅助色使用青蓝和黄绿色，只用于状态、标签和 hover，不做单一紫色主题。
- 圆角控制在 8px 内，符合项目 UI 规则。

## 4. 交互和可访问性

- 所有可点击区域至少 44px 高。
- 图标来自 Lucide，不使用 emoji 做结构图标。
- 横向货架只在桌面和平板中作为内容浏览方式；移动端改为单列，避免页面级横向滚动。
- focus ring、disabled、loading 状态保留。
- 创建表单继续保留可见 label、错误提示和可见性切换。

## 5. 不做

- 不添加假 Profile、假社交入口、假作者数据或假作品。
- 不改数据库结构和生成链路。
- 不把详情页 iframe 改回 Blob URL，继续使用同源代理播放。
- 不做评论、通知、创作者经济、搜索等超出 MVP 的平台能力。

## 6. 验证

完成后需要运行：

- `npx prisma generate`
- `npm run lint`
- `npm run build`

涉及 UI，还需要启动本地服务检查：

- 首页桌面：左侧导航、内容货架、卡片浮层不遮挡。
- 首页移动：无页面级横向滚动，底部导航不遮挡主要操作。
- 创建页：prompt 输入、可见性切换、错误提示可用。
- 详情页：游戏舞台优先，操作面板在桌面右侧、移动端下方。
